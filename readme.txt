## ✨ Возможности

### 🖼️ Оптимизация изображений
- Автоматическое сжатие при загрузке
- Конвертация в формат WebP
- Оптимизация существующих изображений в библиотеке
- Поддержка JPEG, PNG, GIF, WebP
- Сохранение прозрачности PNG
- Оптимизация всех размеров миниатюр

### 🗄️ Оптимизация базы данных
- Очистка ревизий записей
- Удаление записей в корзине
- Очистка транзиентов (временных данных)
- Удаление спам-комментариев
- Очистка orphaned мета-данных
- Оптимизация таблиц MySQL

### 📊 Мониторинг здоровья сайта
- Проверка версии PHP и WordPress
- Мониторинг использования памяти
- Контроль дискового пространства
- Проверка безопасности
- Статус кэширования
- Мониторинг WP-Cron

### ⚡ Автоматизация
- Ежедневная автоматическая оптимизация
- Еженедельная проверка здоровья
- Уведомления о критических проблемах
- Быстрая оптимизация из админ-бара

## 📦 Требования

### Минимальные
- WordPress 5.0 или выше
- PHP 7.2 или выше
- MySQL 5.6 или выше / MariaDB 10.1 или выше
- Расширение GD

### Рекомендуемые
- PHP 8.0+
- Расширение Imagick (для лучшей оптимизации изображений)
- Redis или Memcached (для объектного кэширования)
- Минимум 256MB памяти

## 🔧 Установка

### Автоматическая установка
1. Скачайте архив с плагином
2. В админ-панели WordPress перейдите в **Плагины → Добавить новый**
3. Нажмите **Загрузить плагин** и выберите архив
4. Активируйте плагин

### Ручная установка
1. Скопируйте папку `site-optimizer` в `/wp-content/plugins/`
2. Активируйте плагин в меню **Плагины**

### Через WP-CLI
```bash
wp plugin install site-optimizer --activate
```

## 📖 Использование

### Быстрый старт

1. После активации перейдите в **Инструменты → Site Optimizer**
2. Проверьте статус сайта на вкладке **Здоровье**
3. Запустите оптимизацию изображений
4. Выполните очистку базы данных

### Оптимизация изображений

#### Автоматическая оптимизация
Изображения оптимизируются автоматически при загрузке через медиа-библиотеку.

#### Ручная оптимизация всех изображений
1. Перейдите в **Инструменты → Site Optimizer → Изображения**
2. Нажмите **Оптимизировать все изображения**
3. Дождитесь завершения процесса

#### Создание WebP
WebP версии создаются автоматически для всех оптимизированных изображений.

### Оптимизация базы данных

1. Перейдите в **Инструменты → Site Optimizer → База данных**
2. Выберите тип очистки или нажмите **Полная очистка**
3. Подтвердите действие

> ⚠️ **Важно:** Перед оптимизацией базы данных рекомендуется сделать резервную копию!

### Мониторинг здоровья

1. Перейдите в **Инструменты → Site Optimizer → Здоровье**
2. Просмотрите статус всех компонентов
3. Исправьте найденные проблемы

## ⚙️ Настройки

Плагин использует следующие опции (хранятся в `wp_options`):

### so_settings
```php
array(
    'enable_image_optimization' => true,     // Включить оптимизацию изображений
    'enable_webp' => true,                   // Включить создание WebP
    'enable_database_cleanup' => true,       // Включить очистку БД
    'enable_cron' => true,                   // Включить Cron задачи
    'image_quality' => 85,                   // Качество сжатия (1-100)
    'auto_optimize_on_upload' => true        // Авто-оптимизация при загрузке
)
```

### so_stats
```php
array(
    'images_optimized' => 0,      // Всего оптимизировано изображений
    'space_saved' => 0,           // Сэкономлено места (байт)
    'optimizations_run' => 0,     // Количество запусков оптимизации
    'db_optimizations' => 0,      // Количество оптимизаций БД
    'last_optimization' => 0,     // Время последней оптимизации
    'last_health_check' => 0      // Время последней проверки здоровья
)
```

## 🎯 Хуки и API

### Cron хуки
```php
// Ежедневная оптимизация
add_action('so_daily_optimization', 'your_function');

// Еженедельная проверка здоровья
add_action('so_weekly_health_check', 'your_function');
```

### AJAX действия
```php
// Оптимизация изображений
do_action('wp_ajax_so_optimize_images');

// Оптимизация базы данных
do_action('wp_ajax_so_optimize_database');

// Получение статуса здоровья
do_action('wp_ajax_so_get_health_status');

// Очистка транзиентов
do_action('wp_ajax_so_purge_transients');

// Быстрая оптимизация
do_action('wp_ajax_so_quick_optimize');
```

### Статические методы классов

```php
// Оптимизация изображений
SO_Image_Optimizer::optimize_all_images();
SO_Image_Optimizer::optimize_image($file_path);
SO_Image_Optimizer::create_webp($file_path);
SO_Image_Optimizer::get_stats();

// База данных
SO_Database_Optimizer::full_cleanup();
SO_Database_Optimizer::optimize_tables();
SO_Database_Optimizer::clean_post_revisions();
SO_Database_Optimizer::clean_trash();
SO_Database_Optimizer::clean_transients();
SO_Database_Optimizer::get_stats();

// Здоровье сайта
SO_Site_Health::get_full_status();
SO_Site_Health::check_php_version();
SO_Site_Health::check_memory_usage();
SO_Site_Health::check_disk_space();
```

## ❓ Частые вопросы

### Безопасно ли удалять ревизии записей?
Да, ревизии нужны только для истории изменений. Рекомендуется сохранять последние 3-5 ревизий.

### Можно ли отключить создание WebP?
Да, через фильтр:
```php
add_filter('so_enable_webp', '__return_false');
```

### Как исключить определённые изображения из оптимизации?
Используйте фильтр:
```php
add_filter('so_skip_image_optimization', function($skip, $attachment_id) {
    $meta = wp_get_attachment_metadata($attachment_id);
    return ($meta['filesize'] < 10240); // Пропускать файлы < 10KB
}, 10, 2);
```

### Плагин замедляет загрузку изображений?
Нет, оптимизация происходит при загрузке файла. Оптимизированные изображения загружаются быстрее.

### Как часто запускать оптимизацию БД?
Рекомендуется раз в неделю для активных сайтов, раз в месяц — для малоактивных.

## 🐛 Решение проблем

### Ошибка "Недостаточно памяти"
Увеличьте лимит памяти в `wp-config.php`:
```php
define('WP_MEMORY_LIMIT', '256M');
```

### Не создаются WebP файлы
Проверьте наличие расширения GD с поддержкой WebP:
```php
var_dump(function_exists('imagewebp'));
```

### Ошибка оптимизации базы данных
Проверьте права пользователя MySQL на выполнение `OPTIMIZE TABLE`.
